我这两天准备字节 coding 面试快累死了，但是找到个保证不偷懒的方法。

流程是这样：

1. **自己先读题，先做**。能做出来，这题还行；做不出来，大概率是卡住了，就去问 Claude。不让它直接给代码，先给思路，我根据思路尝试一点点做。
2. **做出来就结束；做不出来，再让它给代码**，写一遍并分析为什么这样写。
3. **弄完后做两件事**：在备忘录里把代码粘贴下来，删掉刚才写的部分，自己重写，再提交到 LeetCode 上。哪里错改哪里，提交好了后在 LeetCode 上再重写一遍。
4. **写个复盘**，这样感觉比较扎实。

核心逻辑就是：先暴露自己不会的地方，然后补，最后通过"删掉重写"强制自己不靠记忆照抄。
